# 🚀 德育积分系统上线执行清单

快速参考版，一页纸涵盖所有步骤。详细说明见 `CLOUD_DEPLOYMENT_AND_LAUNCH_GUIDE.md`

---

## 📋 第一阶段：基础设施准备（1-2 周）

### 周一（购买）

- [ ] 选择云厂商（阿里云、腾讯云、华为云）
- [ ] 购买云服务器（2 核 4G 推荐）
  - [ ] 记录：服务器 IP、用户名、密码
- [ ] 购买域名（.com 或 .cn）
  - [ ] 记录：域名名称、账号密码
- [ ] 申请微信小程序账号
  - [ ] 记录：AppID、AppSecret

### 周二-周三（配置网络）

- [ ] 域名解析到服务器 IP
  - [ ] A 记录：@ → 服务器 IP
  - [ ] 等待 DNS 生效（5-10 分钟）
- [ ] 配置云服务器安全组
  - [ ] 允许 22 端口（SSH）
  - [ ] 允许 80 端口（HTTP）
  - [ ] 允许 443 端口（HTTPS）
  - [ ] 允许 3000 端口（应用，暂时）
- [ ] 申请 SSL 证书（自动 HTTPS）
  - [ ] 使用免费 Let's Encrypt
  - [ ] 或使用云厂商免费证书

### 周四-周五（备案）

- [ ] 通过云厂商开始备案流程
  - [ ] 填写网站信息
  - [ ] 上传身份证和营业执照
  - [ ] 拍照留念（提供背景布）
- [ ] 等待审核
  - [ ] 云厂商初审：1-3 天
  - [ ] 工信部审核：5-20 天
  - [ ] **总计：2-4 周**（备案可与代码部署并行）

---

## 🔧 第二阶段：代码部署（1-2 周）

### 周一（环境配置）

- [ ] SSH 连接到服务器
- [ ] 更新系统：`sudo apt update && sudo apt upgrade -y`
- [ ] 安装 Node.js：`curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -`
- [ ] 安装 MySQL：`sudo apt install -y mysql-server`
- [ ] 安装 Nginx：`sudo apt install -y nginx`
- [ ] 安装 PM2：`sudo npm install -g pm2`

### 周二（部署代码）

- [ ] 创建应用目录：`mkdir -p /home/ubuntu/apps`
- [ ] 上传项目代码（SCP 或 Git 克隆）
- [ ] 安装依赖：`npm install`
- [ ] 配置 `.env` 文件（数据库、JWT 密钥等）
- [ ] 初始化数据库：
  ```bash
  mysql -u root -p < database.sql
  ```
- [ ] 使用 PM2 启动：`pm2 start server.js --name "score-api"`

### 周三（配置 Nginx 和 HTTPS）

- [ ] 配置 Nginx 反向代理
  - [ ] 创建配置文件
  - [ ] 配置 SSL 证书路径
  - [ ] 指向后端应用（localhost:3000）
- [ ] 启用 HTTPS：`sudo systemctl reload nginx`
- [ ] 测试 HTTPS：`curl https://yourdomain.com`（应显示证书信息）

### 周四-周五（测试）

- [ ] 修改小程序后端地址为云服务器地址
- [ ] 测试后端 API：
  ```bash
  curl -X POST https://api.yourdomain.com/api/auth/teacher/login \
    -H "Content-Type: application/json" \
    -d '{"username":"teacher1","password":"teacher1"}'
  ```
- [ ] 测试管理后台：打开浏览器访问 `https://admin.yourdomain.com`
- [ ] 测试小程序：
  - [ ] 登录
  - [ ] 查看学生列表
  - [ ] 添加积分
  - [ ] 导出数据
- [ ] 检查性能和错误日志

---

## 📱 第三阶段：小程序上线（1-2 周）

### 周一（准备）

- [ ] 准备小程序信息
  - [ ] 名称、简介、分类
  - [ ] 上传 512x512 图标
  - [ ] 编写隐私政策
  - [ ] 编写用户协议
- [ ] 小程序后台配置
  - [ ] 添加服务器域名到白名单
  - [ ] 配置服务器地址
  - [ ] 启用测试账号

### 周二（测试）

- [ ] 在微信开发者工具中上传开发版本
- [ ] 邀请测试员进行内测
  - [ ] 分享开发版体验码
  - [ ] 收集反馈
  - [ ] 修复问题

### 周三-周四（提交审核）

- [ ] 更新版本号
- [ ] 再次全面测试（无崩溃、无错误）
- [ ] 提交正式审核
  - [ ] 填写功能描述（准确、完整）
  - [ ] 提供有效的测试账号
  - [ ] 确保隐私政策和用户协议有效
  - [ ] 点击「提交」

### 周五-周六（等待审核）

- [ ] 监控审核进度
- [ ] 如被拒，阅读原因，修改后重新提交
- [ ] 通常 3-7 天内得到结果

### 周七（上线）

- [ ] 审核通过后点击「发布」
- [ ] 等待 10 分钟，小程序上线
- [ ] 测试在微信中搜索小程序
- [ ] 生成小程序码用于推广

## ✅ 上线前最终检查

### 后端检查

- [ ] 数据库已迁移到云服务器
- [ ] 所有 API 都返回正确的响应
- [ ] 错误处理正确（无 500 错误）
- [ ] 日志记录正常
- [ ] 性能可接受（响应时间 < 1 秒）
- [ ] SSL 证书有效（浏览器显示🔒）
- [ ] 支持 CORS（小程序可跨域调用）
- [ ] 数据备份已启用

### 管理后台检查

- [ ] 所有功能可用（班级、班主任、学生、积分、学期）
- [ ] 导入功能正常
- [ ] 导出功能正常
- [ ] 错误提示清晰

### 小程序检查

- [ ] 登录功能正常
- [ ] 学生列表可加载
- [ ] 积分操作可执行
- [ ] 数据导出可使用
- [ ] 没有明显的 UI 错误
- [ ] 性能流畅（加载快、操作响应快）
- [ ] 没有权限提示

### 安全检查

- [ ] HTTPS 已启用
- [ ] 管理员密码强度足够
- [ ] 数据库访问仅限内网
- [ ] 敏感信息不在代码中
- [ ] 定期备份机制已启动

### 文档检查

- [ ] 隐私政策有效且完整
- [ ] 用户协议有效且完整
- [ ] 服务条款清晰
- [ ] 使用说明已准备

## 🚩 风险预案

| 风险 | 概率 | 应对 |
|-----|-----|-----|
| 备案延长 | 中 | 与代码部署并行进行 |
| 小程序审核拒绝 | 低 | 修改后重新提交（通常快速通过） |
| 域名配置失败 | 低 | 使用 IP 临时访问，然后修复 DNS |
| 性能不达预期 | 低 | 升级服务器配置或优化代码 |
| 数据库迁移失败 | 很低 | 使用备份恢复，重新导入 |
