## 🎯 常见场景速查

### 场景1：新部署系统

```bash
# 1. 初始化数据库
mysql -u root -p < database.sql

# 2. 启动后端
npm start

# 3. 访问管理后台
浏览器打开 http://localhost:3000/admin
用户名: admin
密码: (查看数据库初始化)
```

### 场景2：升级现有系统（保留班主任数据）

```bash
# 执行迁移脚本修改外键策略
node migrate_score_records_keep_on_teacher_delete.js
```

### 场景3：添加班级

**方法 1：单个添加**
- 管理后台 → 班级管理 → 新增班级 → 输入名称

**方法 2：批量添加**
- 管理后台 → 班级管理 → 批量添加
  - 前缀：`2026`
  - 起始号：`01`
  - 结束号：`05`
  - 生成：202601班 ～ 202605班

### 场景4：导入学生

1. 管理后台 → 学生管理 → 导入学生
2. 选择 Excel 文件（格式：姓名、学号、班级、性别）
3. 查看导入结果和错误详情

### 场景5：导出积分数据

1. 管理后台 → 积分数据
2. 可选导出类型：
   - **积分记录**（包含所有操作）
   - **学生总分**（整数显示）
3. 点击「导出Excel」

## 📋 系统管理员检查清单

### 日常维护

- [ ] 每周备份数据库一次
- [ ] 检查操作日志是否异常
- [ ] 确认激活的学期是否正确
- [ ] 查看系统日志 `/uploads` 文件夹

### 学期更换

1. 创建新学期（学期管理 → 新增学期）
2. 激活新学期（点击「激活」按钮）
3. 导出上学期数据作为存档
4. （可选）重置学生积分

### 班级调整

- 批量添加班级：批量添加 → 输入前缀和号码范围
- 删除班级：类名下无班主任和学生时可删除
- 导入学生时务必先创建班级

### 用户管理

- 添加班主任：班主任管理 → 新增班主任 → 填写信息
- 删除班主任：点击「删除」按钮（积分记录保留）
- 重置密码：直接修改数据库或重建用户

## 🔍 故障排查

### 问题：后端无法启动

```bash
# 检查 3000 端口是否被占用
netstat -ano | findstr :3000

# 杀死占用进程
.\kill-port-3000.ps1

# 重新启动
npm start
```

### 问题：数据库连接失败

```bash
# 检查 .env 配置
cat .env

# 验证 MySQL 服务运行
mysql -u root -p -e "SELECT 1;"
```

### 问题：小程序登录失败

- 检查后端是否运行：`npm start`
- 检查小程序中 API 地址是否正确
- 查看浏览器控制台错误信息

### 问题：导入学生出错

- 检查 Excel 列顺序：姓名、学号、班级、性别
- 确认班级已在系统中创建
- 查看详细错误提示信息

## 💡 提示与最佳实践

### ✅ 建议做法
- 每次系统更新后执行迁移脚本
- 定期备份数据库（至少每周一次）
- 新学期开始时激活新学期
- 使用批量导入提高效率

### ❌ 避免做法
- 不要直接修改 `students` 表的 `current_score`（会导致不一致）
- 不要删除有班主任或学生的班级
- 不要同时激活多个学期
- 不要在小程序和管理后台同时编辑数据
