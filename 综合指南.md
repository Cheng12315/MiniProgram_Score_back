## 系统概述

德育积分管理系统是一套完整的学生德育积分管理解决方案，包含：
- **管理后台**（Web）：管理员进行班级、班主任、学生、学期、积分数据的管理
- **微信小程序**：班主任查看和录入学生积分
- **Node.js 后端**：提供 RESTful API，使用 MySQL 数据库

**技术栈：**
- 后端：Express.js + MySQL2
- 管理后台：HTML5 + CSS3 + Vanilla JavaScript
- 小程序：微信小程序原生框架

## 快速开始

### 前置要求
- Node.js (v14+)
- MySQL (v5.7+)
- 已安装项目依赖 (`npm install`)

### 启动后端服务

```bash
cd D:\ScoreManagement_back
npm start
```

**默认配置：**
- 后端地址：`http://localhost:3000`
- 数据库：根据 `.env` 文件配置（默认 localhost:3306）

**注意：** 若 3000 端口被占用，运行：
```powershell
.\kill-port-3000.ps1
```

### 访问管理后台

打开浏览器访问(无痕模式，ctrl+shift+n)：
```
http://localhost:3000/admin
```

**默认管理员账号：** `admin` / 密码见数据库初始化文档

### 启动小程序

使用微信开发者工具打开 `D:\minipro_ScoreManagement` 目录

## 部署指南

### 1. 数据库初始化

#### 首次部署（新库）

```bash
# 运行初始化脚本
mysql -u root -p < database.sql
```

**初始内容包括：**
- 8 个表（班级、班主任、管理员、学期、学生、积分记录等）
- 默认学期（2024春、2024秋、2025春）
- 默认班级（8年级1班～8年级2班、242班、243班）

#### 现有环境迁移（删除班主任时保留积分记录）

若系统已运行但需升级外键策略，执行迁移脚本：

```bash
node migrate_score_records_keep_on_teacher_delete.js
```

**作用：** 将 `score_records.teacher_id` 外键从 `ON DELETE CASCADE` 改为 `ON DELETE SET NULL`
- 删除班主任时，其积分记录保留，操作人显示为「已删除」
- 旧的外键会被删除并重新创建

### 2. 环境配置

编辑 `.env` 文件：

```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=score_management
JWT_SECRET=your_secret_key
UPLOAD_DIR=uploads
NODE_ENV=production
```

### 3. 启动服务

```bash
npm start
```

服务启动后输出：
```
✓ MySQL 数据库连接成功
✓ 服务器运行在 http://localhost:3000
```

### 4. 备份与恢复

**备份数据库：**
```bash
mysqldump -u root -p score_management > backup_$(date +%Y%m%d_%H%M%S).sql
```

**恢复数据库：**
```bash
mysql -u root -p score_management < backup_20260302_120000.sql
```

## 管理后台使用手册

### 登录

访问 `http://localhost:3000/admin`，输入管理员账号密码登录。

### 功能模块

#### 1. 仪表板
- 实时统计：学期数、班级数、班主任数、学生数、积分记录数
- 最近操作日志显示

#### 2. 班主任管理
**列表操作：**
- 查看、搜索班主任
- 删除班主任（积分记录保留）

**新增班主任：**
1. 点击「新增班主任」按钮
2. 填写：用户名、密码（明文显示）、真实姓名、班级
3. 点击「创建」

**说明：** 密码在表单中以明文显示便于验证

#### 3. 学生管理

**导入学生：**
1. 点击「导入学生」按钮，选择 Excel 文件
2. 文件格式要求（三列）：
   ```
   姓名      学号    班级  性别
   张三      001    8年级1班  男
   李四      002    8年级1班  女
   ```
3. 导入成功或失败时，显示详细错误信息（逐行）

**说明：** 
- 班级必须先在「班级管理」中创建
- 学号重复时自动更新该学生
- 缺少必需字段或班级不存在时会报错

#### 4. 学期管理

**操作：**
- 新增学期：输入名称、开始日期、结束日期
- 激活学期：点击「激活」按钮，只能同时激活一个学期
- 激活学期会作为小程序的默认选中学期

#### 5. 班级管理

**单个添加：**
1. 点击「新增班级」，输入班级名
2. 如：「8年级1班」、「663班」等

**批量添加：**
1. 点击「批量添加」
2. 填写：
   - **前缀**（可留空）：如「2026」
   - **起始号**：如「01」（支持补零）
   - **结束号**：如「05」
3. 示例：
   - 前缀「2026」+起始「01」+结束「05」 → 202601班～202605班
   - 前缀留空+起始「663」+结束「730」 → 663班～730班

#### 6. 积分数据

**查看积分记录：**
- 按班级、学期、学生名/学号筛选
- 分页显示，按操作时间倒序

**导出 Excel：**
1. 选择导出类型：
   - **积分记录**：包含学号、姓名、班级、变动原因、分值、操作时间、操作人
   - **学生总分**：包含学号、姓名、班级、得分（整数显示）
2. 点击「导出Excel」

**说明：** 操作人为「已删除」表示该班主任已被删除

#### 7. 操作日志

查看所有管理员操作的历史记录。

## 小程序使用手册

### 功能模块

#### 1. 登录
- 输入班主任用户名和密码
- 登录失败时显示明确提示：「账号或密码错误」
- Token 过期时自动跳转到登录页

#### 2. 学生列表

**页面功能：**
- 选择学期（默认选中已激活学期）
- 搜索学生（按学号或姓名）
- 查看班级下的所有学生

**操作：**
- 点击学生卡片可进入「积分操作」页面
- 点击「导入」可批量导入学生

#### 3. 积分操作

**添加积分：**
1. 选择学生
2. 输入变动理由（如「校运会一等奖」）
3. 输入分值（正数为加分，负数为扣分）
4. 提交

**说明：** 分值直接反映在学生当前学期的积分中

#### 4. 积分查询

- 查看学生在各学期的积分变动历史
- 支持按学期筛选
- 撤销功能：删除已录入的积分记录

#### 5. 数据导出

**导出自班级数据：**
- 按学期导出班级内所有学生的积分记录
- 或导出学生总分汇总

## 常见问题

### Q1: 删除班主任后，他的积分记录消失了？

**A:** 这是旧版本的行为（外键级联删除）。

**解决方案：**
- 执行迁移脚本升级数据库：
  ```bash
  node migrate_score_records_keep_on_teacher_delete.js
  ```
- 之后删除班主任时，积分记录会保留，操作人显示为「已删除」

### Q2: 导入学生时出现错误，如何查看具体是哪一行？

**A:** 导入完成后会弹窗显示所有错误信息，格式如下：
```
第3行: 班级"未知班级"不存在
第5行: 缺少必需字段（需要：姓名、学号、班级、性别）
```

### Q3: 批量添加班级时，补零不生效（如「01」变成「1」）？

**A:** 请使用文本形式（不是数字）输入起始号和结束号。管理后台已改为文本输入框。

### Q4: 学生总分导出时，为什么显示小数（如 100.00）？

**A:** 已修复为整数显示（如 100）。

### Q5: 小程序登录失败提示「HTTP Error: 401」？

**A:** 已修复为更明确的提示「账号或密码错误」。

### Q6: 如何重置管理员密码？

**A:** 执行 SQL：
```sql
UPDATE admins SET password = '新密码哈希值' WHERE username = 'admin';
```

或使用项目中的重置脚本（需输入新密码）。

## API 文档

### 认证 API

#### 班主任登录
```
POST /api/auth/teacher/login
Content-Type: application/json

{
  "username": "teacher1",
  "password": "password123"
}

响应:
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "token": "eyJ...",
    "user": {
      "id": 1,
      "username": "teacher1",
      "real_name": "张老师",
      "class_id": 1,
      "class_name": "8年级1班"
    }
  }
}
```

#### 管理员登录
```
POST /api/auth/admin/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}
```

### 班主任管理 API

#### 获取班主任列表
```
GET /api/admin/teachers?page=1&pageSize=10&search=
Authorization: Bearer {token}
```

#### 创建班主任
```
POST /api/admin/teachers
Authorization: Bearer {token}
Content-Type: application/json

{
  "username": "teacher2",
  "password": "pass123",
  "realName": "李老师",
  "classId": 2
}
```

#### 删除班主任
```
DELETE /api/admin/teachers/{id}
Authorization: Bearer {token}
```

### 学生管理 API

#### 获取学生列表
```
GET /api/admin/students?page=1&pageSize=10&classId=1&search=
Authorization: Bearer {token}
```

#### 导入学生
```
POST /api/admin/students/import
Authorization: Bearer {token}
Content-Type: multipart/form-data

File: <Excel file>
```

### 学期管理 API

#### 获取学期列表
```
GET /api/admin/semesters
Authorization: Bearer {token}
```

响应中 `is_active: true` 的学期是当前激活学期。

#### 创建学期
```
POST /api/admin/semesters
Authorization: Bearer {token}
Content-Type: application/json

{
  "semester_name": "2025秋",
  "start_date": "2025-09-01",
  "end_date": "2026-01-20"
}
```

#### 激活学期
```
PATCH /api/admin/semesters/{id}/activate
Authorization: Bearer {token}
```

### 班级管理 API

#### 获取班级列表
```
GET /api/admin/classes
Authorization: Bearer {token}
```

#### 创建班级
```
POST /api/admin/classes
Authorization: Bearer {token}
Content-Type: application/json

{
  "className": "8年级3班"
}
```

#### 批量创建班级
```
POST /api/admin/classes/batch
Authorization: Bearer {token}
Content-Type: application/json

{
  "prefix": "2026",
  "start": "01",
  "end": "05"
}

响应: 生成的班级数组
```

### 积分数据 API

#### 获取积分记录
```
GET /api/admin/score-records?page=1&pageSize=10&classId=1&semesterId=1&search=
Authorization: Bearer {token}
```

#### 导出学生总分
```
GET /api/admin/export/students?classId=1&semesterId=1
Authorization: Bearer {token}
```

#### 导出积分记录
```
GET /api/admin/export/records?classId=1&semesterId=1
Authorization: Bearer {token}
```
