## 数据流向图

### 1. 班主任登录流程

```
┌──────────────┐
│ 小程序登录页  │
└───────┬──────┘
        │ 输入用户名、密码
        ▼
┌──────────────────────────┐
│ POST /api/auth/teacher/login
│ {username, password}     │
└───────┬──────────────────┘
        │
        ▼
┌──────────────────────────────────────┐
│ 后端：查询 teachers 表                │
│      验证密码（bcrypt）              │
│      生成 JWT Token                 │
└───────┬──────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────┐
│ {token, userInfo}                   │
│ 返回给小程序                         │
└───────┬──────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────┐
│ 小程序保存 Token 和 userInfo          │
│ 本地存储                             │
└──────────────────────────────────────┘
```

### 2. 积分操作流程

```
┌─────────────────────────┐
│ 班主任点击学生进行积分   │
│ 输入：分值、理由         │
└────────┬────────────────┘
         │
         ▼
┌──────────────────────────────┐
│ POST /api/teacher/score-records
│ {studentId, score, reason}  │
│ Header: {token}             │
└────────┬────────────────────┘
         │
         ▼
┌────────────────────────────────────────┐
│ 后端：验证 Token（JWT）                 │
│      检查学生是否属于该班级（权限）   │
│      启动事务                         │
└────────┬───────────────────────────────┘
         │
         ├─ INSERT score_records
         │  (student_id, score_change, reason...)
         │
         └─ UPDATE students
            SET current_score = current_score + score
            WHERE id = student_id
         │
         ▼
┌────────────────────────────────────────┐
│ 提交事务                               │
│ 返回：{recordId, newScore}             │
└────────┬───────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────┐
│ 小程序显示操作成功                    │
│ 刷新学生列表和记录                    │
└────────────────────────────────────────┘
```

### 3. 数据导入流程

```
┌──────────────────────────┐
│ 班主任选择 Excel 文件    │
│ 点击"导入学生"按钮       │
└────────┬─────────────────┘
         │
         ▼
┌──────────────────────────────────┐
│ POST /api/teacher/students/import
│ Content-Type: multipart/form-data
│ file: [Excel 文件]               │
└────────┬─────────────────────────┘
         │
         ▼
┌────────────────────────────────────────┐
│ 后端处理：                             │
│ 1. 使用 xlsx 库读取 Excel 文件        │
│ 2. 逐行验证数据（学号、性别等）      │
│ 3. 启动事务                          │
│ 4. 遍历每一行：                       │
│    - 检查学号是否已存在               │
│    - 存在则 UPDATE，不存在则 INSERT  │
│ 5. 提交事务                          │
│ 6. 删除上传的文件                     │
└────────┬───────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────┐
│ 返回：{successCount, errors}          │
│ 显示导入结果                          │
└────────────────────────────────────────┘
```

### 4. 数据导出流程

```
┌──────────────────────┐
│ 班主任点击"导出总分" │

└────────┬───────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────┐
│ 返回 Excel 文件给客户端                │

└────────┬───────────────────────────────┘

         ▼
```

## 事务处理示例

### 积分操作的事务流程

```
用户提交积分操作
        │
        ▼
BEGIN TRANSACTION
        │
        ├─ INSERT INTO score_records
        │  (student_id, score_change, ...)
        │  ✓ 成功 / ✗ 失败
        │
        ├─ UPDATE students
        │  SET current_score = current_score + score_change
        │  WHERE id = student_id
        │  ✓ 成功 / ✗ 失败
        │
        ├─ 如果都成功 ──→ COMMIT
        │                └─→ 返回 200 成功响应
        │
        └─ 如果任意失败 ──→ ROLLBACK
                         └─→ 返回 500 错误响应
```

## 文件上传流程

```
用户选择 Excel 文件
        │
        ▼
前端发送 POST 请求 (multipart/form-data)
        │
        ▼
multer 中间件接收文件
        │
        ├─ 文件验证（后缀名）
        │
        ├─ 保存到 uploads 目录
        │
        └─ req.file 包含文件信息
        │
        ▼
后端逻辑处理
        │
        ├─ xlsx 库读取 Excel
        │
        ├─ 逐行验证数据
        │
        ├─ 数据库操作（INSERT/UPDATE）
        │
        └─ 删除上传的文件
        │
        ▼
返回处理结果
        │
        ├─ successCount: {added, updated}
        ├─ errors: [错误信息数组]
        └─ hasErrors: true/false
```

## 密码加密流程

```
用户输入密码 "password123"
        │
        ▼
    ┌───────────────────────┐
    │ bcryptjs.hash()       │
    │ (salt rounds: 10)     │
    │ 生成哈希值            │
    └────────┬──────────────┘
             │
             ▼
    "$2b$10$..."  (60字符哈希值)
             │
             ├─ 存储到数据库
             │
             └─ 比对时使用 bcrypt.compare()
                输入密码 vs 存储哈希值
                return true/false
```

## 扩展性考虑

### 当前支持规模
- 班级数：无限制
- 班主任数：无限制
- 学生数：100,000+（根据索引优化）
- 操作记录：无限制

### 索引优化
所有表都已添加以下索引：
- 班级表：class_name
- 班主任表：username, class_id
- 学生表：class_id, name, student_number
- 积分记录表：student_id, semester_id, class_id, teacher_id, created_at
- 操作日志表：admin_id, created_at

### 性能优化建议
1. 数据库连接池优化
2. Redis 缓存（学期列表、班级列表）
3. CDN 加速（静态资源）
4. 数据库分片（超大规模时）
