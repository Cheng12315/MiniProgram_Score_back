```
Day 1：购买
├─ 云服务器（获得 IP）

├─ 上传项目代码
├─ 安装依赖
└─ 初始化数据库

└─ 全链路测试

### 第 3-4 周：小程序上线

```

## 🎯 三种上线方案

### 方案 A：极速上线（最快，13 天）

**条件：** 不等备案，使用临时方案

```
Day 1-3：购买和基础配置
Day 4-7：代码部署（使用 IP 临时访问）
Day 8-13：小程序审核和上线
Day 14+：备案完成后，切换域名访问

风险：
  - 备案期间无法通过域名访问
  - 小程序提交审核时后端地址不稳定
  
优点：
  - 快速验证系统可用性
  - 让用户尽早使用
```

**不推荐**（不稳定）

### 方案 B：推荐方案（4-6 周）

**条件：** 备案和部署并行进行

```
Week 1：
  ├─ Day 1：购买云服务器 + 域名 + 申请小程序
  ├─ Day 2：配置网络和 DNS
  ├─ Day 3：开始备案流程
  ├─ Day 3：启动代码部署
  └─ Day 7：部署完成

Week 2-4：
  ├─ 继续等待备案（1-3 周）
  ├─ 代码测试和优化
  ├─ 小程序上传审核
  └─ 小程序审核过程中备案完成

Week 5-6：
  ├─ 备案完成，切换域名访问
  ├─ 小程序审核通过
  └─ 正式上线
```

**优点：**
  - 稳定、有序
  - 上线时所有基础设施完整
  - 有充分的测试时间

**推荐使用此方案 ✓**

### 方案 C：企业级部署（6-8 周）

**条件：** 全部完成后再上线

```
Week 1-2：云基础设施完成
  └─ 购买、配置、备案

Week 2-3：代码部署与优化
  ├─ 完整的性能测试
  ├─ 安全审计
  └─ 高可用配置

Week 4-5：小程序准备
  ├─ 内测（1 周以上）
  ├─ 集中反馈修改
  └─ 提交审核

Week 6：等待审核

Week 7-8：正式上线
```

**优点：**
  - 最稳定、最专业
  - 上线前彻底验证
  - 风险最低

**成本：** 与方案 B 基本相同，主要差别是时间和细致程度。
